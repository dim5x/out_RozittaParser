# Как помочь Rozitta Parser

Спасибо, что хотите внести вклад!

## Что нужно сделать прямо сейчас

Самые важные задачи — **критические баги** из раздела «Архитектура» (`ARCHITECTURE.md`).

## Как внести изменения

1. **Создайте Issue** (если его ещё нет) или найдите существующее.
2. **Назначьте на себя** (если вы участник) или напишите комментарий «Беру».
3. **Вносите правки** в код (следуйте `CLAUDE.md`).
4. **Создайте Pull Request** — даже если не уверены, можно обсудить.

## Сборка и тестирование

- Для разработки достаточно запустить `python main.py` из исходников (инструкция в `README.md`).
- Бинарные сборки делаются через GitHub Actions (кнопка **Run workflow**).
- **Не пытайтесь собрать .exe локально** — это сложно и не нужно.

## Стиль кода

Примерно PEP 8, но главное — не ломайте то, что работает.  
Если сомневаетесь, посмотрите на соседние файлы.

Задавайте в Issue или в обсуждениях
